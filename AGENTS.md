## 正本と参照優先順位

- `docs/` は仕様の正本
- `config/ddl/common/` と `config/ddl/tenant/` は DDL の正本
- 実装前に `docs/` と DDL を確認すること
- `docs/` / DDL / 実装に不整合があれば、実装前に差分として報告すること

## 実装の禁止事項

- 推測で新テーブル、新カラムを追加しないこと
- 既存 DDL を勝手に変更しないこと
- DDL 未定義のテーブルをコードから参照しないこと
- `docs/` や DDL の変更が必要な場合は、先に理由を示すこと

## 維持する絶対条件（変更禁止）

- 満期一覧 = 契約一覧（契約一覧の独立画面を作らない）
- 満期詳細 = 契約詳細（契約詳細の独立画面を作らない）
- ホームは入口画面に徹し、登録・更新・処理を持たせない
- 顧客一覧・顧客詳細は独立画面として維持する
- 事故案件一覧・詳細は main nav とホームの日常業務に含める

## フォルダ責務

- `public/` : Web 入口とエンドポイントのみ
- `src/` : 業務実装の本体
- `docs/patches/` : パッチの一時置き場。本体統合後は速やかに削除する

## 既知の未解消差分（推測で埋めないこと）

- 顧客一覧の列定義が実装と乖離（`docs/screens/customer-list.md §5` の更新が必要）
  - 実装列: 顧客名（リンク） / 満期件数 / 事故件数 / 活動件数 / 最終更新
  - docs 列: 顧客名 / 電話番号 / メール / 担当者 / 契約件数 / 操作（古い定義）
- 監査ログの記録対象が未確定（方針決定後に実装する）
