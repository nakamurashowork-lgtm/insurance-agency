# テナント設定

## 1. 画面概要

### 1-1. 画面名

テナント設定

### 1-2. 画面ID

SCR-TENANT-SETTINGS

### 1-3. 目的

テナント単位で利用する各種設定を管理し、通知動作や業務で利用する選択肢マスタを、運用に合わせて変更できるようにする。

### 1-4. 画面の位置づけ

テナント単位の運用設定を扱う管理画面。  
画面は 2 タブ構成とし、通知設定とマスタ管理を 1 画面内で扱う。

通常業務の主導線には含めず、管理者向けの補助導線またはヘッダーメニュー配下から利用する。  
ダッシュボードから遷移させる場合も、管理者向け補助導線として扱う。

---

## 2. 基本方針

* テナント単位の設定変更を行う画面とする。
* 画面は以下の 2 タブで構成する。
  * 通知設定
  * マスタ管理
* タブは同一画面内の機能として扱い、独立画面には分けない。
* 設定系画面であり、通常業務の主要入口にはしない。
* 主利用者は管理権限を持つ利用者とする。
* 初期段階で参照範囲を広めに取る場合でも、管理画面である位置づけは変えない。
* テナントをまたぐ設定変更はできない。
* 保存対象はテナント単位で明確に分かるようにする。

---

## 3. 想定利用者

* テナント管理者
* 管理者ユーザー
* system_admin 相当の利用者

※ 一般のテナント所属ユーザーは、本画面の主利用者とはしない。  
※ 初期実装で参照可能とする場合でも、通常業務の標準利用画面とは扱わない。

---

## 4. 主な利用シナリオ

### 4-1. 通知設定変更シナリオ

1. 管理権限を持つ利用者がテナント設定を開く。
2. 通知設定タブを表示する。
3. 満期通知や事故リマインドの通知条件を確認する。
4. 必要に応じて通知日数や通知先設定を変更する。
5. 保存後、テナント単位の通知設定として反映する。

### 4-2. マスタ管理シナリオ

1. 管理権限を持つ利用者がテナント設定を開く。
2. マスタ管理タブへ切り替える。
3. 業務で使用する選択肢マスタの一覧を確認する。
4. 必要に応じてマスタ値を追加、更新、削除する。
5. 保存後、対象テナントの業務画面に反映する。

### 5-1. 全体構成

画面は以下のブロックで構成する。

1. ページヘッダー
2. タブエリア

### 5-2. ページヘッダー

| 項目 | 内容 |

### 5-3. タブエリア

| タブ | 内容 |
| --- | --- |

## 6. 通知設定タブ

### 6-1. 目的

満期通知や事故リマインドなど、テナント単位で制御したい通知設定を管理する。

### 6-2. 想定設定項目

| 項目 | 内容 |
| --- | --- |
| 満期通知有効フラグ | 満期通知を有効にするか |
| 満期通知開始日数 | 満期日の何日前から通知するか |
| 満期通知間近日数 | 間近通知を何日前で扱うか |
| 事故リマインド有効フラグ | 事故リマインドを有効にするか |
| 通知先種別 | LINE WORKS などの通知先種別 |
| 通知先情報 | Webhook URL 等の通知先情報 |
| 通知本文設定 | 必要に応じて定型文や件名ルール |

※ 実際の保持項目に応じて調整する。

### 6-3. 主操作

| 操作 | 内容 |
| --- | --- |
| 保存 | 通知設定を保存する |
| 入力値リセット | 変更前の状態へ戻す |

### 6-4. 動作

* 初期表示時に対象テナントの通知設定を表示する。
* 利用者が値を変更して保存すると、対象テナントの設定を更新する。
* 保存成功時はメッセージを表示する。
* 入力不正がある場合は保存しない。

### 6-5. 表示上の考え方

* 満期通知と事故通知をグループ分けして表示する。
* 通知先設定は誤入力しやすいため、ラベルを明確にする。
* 有効フラグがオフの場合、関連項目を非活性にしてよい。

---

## 7. マスタ管理タブ

### 7-1. 目的

テナントごとに管理したい業務選択肢を、画面から追加・更新・削除できるようにする。

### 7-2. 想定管理対象

| マスタ種別 | 内容 |
| --- | --- |
| 種目 | 保険種目などの選択肢 |
| 営業担当者 | 営業担当の選択肢 |
| 事務担当者 | 事務担当の選択肢 |
| 支店 | 支店の選択肢 |
| ステータス候補 | 一覧や編集で使う状態選択肢 |

※ 実際に管理対象とするマスタは実装仕様に合わせて調整する。

### 7-3. 画面構成

マスタ管理タブ内は、管理対象ごとに一覧と編集領域を持つ構成とする。

| 項目 | 内容 |
| --- | --- |
| マスタ種別選択 | 管理したいマスタ種別を選ぶ |
| 一覧表示 | 登録済みマスタ値を一覧表示する |
| 追加入力欄 | 新規値を追加する |
| 編集導線 | 既存値を編集する |
| 削除導線 | 既存値を削除する |
| 表示順 | 必要に応じて並び順を管理する |
| 有効フラグ | 必要に応じて利用可否を管理する |

### 7-4. 主操作

| 操作 | 内容 |
| --- | --- |
| 追加 | 新しいマスタ値を登録する |
| 更新 | 既存のマスタ値を更新する |
| 削除 | 既存のマスタ値を削除する |
| 並び順変更 | 表示順を変更する |

### 7-5. 動作

* 初期表示時に選択中マスタ種別の一覧を表示する。
* 追加、更新、削除後は一覧を再表示する。
* 他画面で利用中の値の扱いは実装方針に従う。
* 削除できない条件がある場合はメッセージ表示する。

### 7-6. 表示上の考え方

* 管理対象が切り替わっても操作方法が大きく変わらないようにする。
* 業務利用頻度の高いマスタは上位に置く。
* 有効フラグ運用がある場合、物理削除より無効化を優先してよい。

| タブ切替 | 通知設定とマスタ管理を切り替える | 主操作 |

| マスタ追加 | 新しいマスタ値を登録する | 主操作 |
| マスタ更新 | 既存マスタ値を更新する | 主操作 |
| マスタ削除 | 既存マスタ値を削除する | 主操作 |
| 戻る | 前画面へ戻る | 補助操作 |

## 9. 画面動作

### 9-1. 初期表示

* ログイン中ユーザーの所属テナントに対応する設定を表示する。
* 初期表示タブは通知設定としてよい。
* テナント名を明示し、どのテナント設定を編集しているか分かるようにする。

### 9-2. タブ切替時

* 選択したタブ内容を表示する。
* 未保存変更がある場合の扱いは実装方針に従う。

### 9-3. 通知設定保存時

* 入力チェック後、対象テナントの通知設定を保存する。
* 保存成功時は成功メッセージを表示する。
* 保存失敗時はエラーメッセージを表示する。

### 9-4. マスタ追加・更新・削除時

* 入力チェック後、対象テナントのマスタを更新する。
* 処理後は一覧を再取得して表示する。
* 他データとの整合上、削除できない場合は理由をメッセージ表示する。

---

## 10. 本画面の役割整理

### 10-1. テナント設定の役割

本画面の役割は、テナント単位で業務運用に関わる設定を変更できるようにすることである。

ここで扱うものは以下である。

* 通知の有効・無効
* 通知条件
* 通知先情報
* 業務で使う選択肢マスタ

### 10-2. ダッシュボードとの違い

ダッシュボードは状況確認と導線の画面である。  
本画面は、運用設定そのものを変更する管理画面である。

### 10-3. 独立画面に分けない理由

通知設定とマスタ管理はどちらもテナント単位の設定であり、利用者視点では同じカテゴリの操作であるため、同一画面内の 2 タブとして扱う。

## 13. 権限・認可

* ログイン済みユーザーのみ表示可能とする。
* 表示・更新対象は所属テナントの設定に限定する。
* 本画面は管理機能として扱い、編集権限は管理権限者に限定することを基本とする。
* 初期実装で参照のみを広く許容する場合でも、更新権限は管理権限者に限定することが望ましい。
* 編集権限がない場合は参照のみ、または画面自体を非表示としてよい。
* 権限差分は通常業務の主導線ではなく、管理機能の公開範囲として扱う。

---

## 14. スマホ表示方針

* タブは押しやすい大きさで表示する。
* 通知設定は縦積みフォームを基本とする。
* マスタ管理は一覧を簡略化し、必要に応じて横スクロールを許容する。
* 保存ボタンは画面下部でも操作しやすい位置に置く。

---

## 15. 例外・考慮事項

### 15-1. 通知先情報が未設定の場合

* 未設定状態を明示する。
* 通知有効化時に必須入力チェックを行ってよい。

### 15-2. マスタ値が他画面で利用中の場合

* 物理削除ではなく無効化を優先してよい。
* 削除不可理由を利用者に表示する。

### 15-3. 複数テナント所属の場合

* 初期仕様では主所属テナントのみ編集対象とする。
* 将来的に切替が必要になった場合は別途検討する。

### 15-4. 未保存状態で画面を離れる場合

* 必要に応じて確認ダイアログを表示してよい。

---

## 16. 非機能要件上の留意点

* テナントをまたぐ更新が起きないこと。
* 通知先情報など誤入力時の影響が大きい項目は、入力補助やバリデーションを設けること。
* マスタ更新後、業務画面への反映タイミングを明確にすること。
* PC とスマホの両方で操作できること。
