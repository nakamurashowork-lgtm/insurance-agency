# テナント設定

## 1. 画面概要

### 1-1. 画面名

テナント設定

### 1-2. 画面ID

SCR-TENANT-SETTINGS

### 1-3. 目的

テナント単位で利用する各種設定を管理し、通知動作や業務で利用する選択肢マスタを、運用に合わせて変更できるようにする。

### 1-4. 画面の位置づけ

テナント単位の運用設定を扱う管理画面。  
画面はタブを使わず、通知設定・マスタ管理・SJNETコード設定を 1 画面内で縦積み表示する。

通常業務の主導線には含めず、管理者向けの補助導線またはヘッダーメニュー配下から利用する。  
ホームから遷移させる場合も、管理・設定として扱う。

---

## 2. 基本方針

* テナント単位の設定変更を行う画面とする。
* 画面は以下の 4 セクションで構成する。
  * テナント概要
  * 通知設定
  * マスタ管理
  * SJNETコード設定
* 同一画面内を縦スクロールして操作し、独立画面には分けない。
* 設定系画面であり、通常業務の主要入口にはしない。
* 主利用者は管理権限を持つ利用者とする。
* 初期段階で参照範囲を広めに取る場合でも、管理画面である位置づけは変えない。
* テナントをまたぐ設定変更はできない。
* 保存対象はテナント単位で明確に分かるようにする。

---

## 3. 想定利用者

* テナント管理者
* 管理者ユーザー
* system_admin 相当の利用者

※ 一般のテナント所属ユーザーは、本画面の主利用者とはしない。  
※ 初期実装で参照可能とする場合でも、通常業務の標準利用画面とは扱わない。

---

## 4. 主な利用シナリオ

### 4-1. 通知設定変更シナリオ

1. 管理権限を持つ利用者がテナント設定を開く。
2. 通知設定セクションを確認する。
3. 満期通知や事故リマインドの通知条件を確認する。
4. 必要に応じて通知日数や通知先設定を変更する。
5. 保存後、テナント単位の通知設定として反映する。

### 4-2. マスタ管理シナリオ

1. 管理権限を持つ利用者がテナント設定を開く。
2. 画面下部へスクロールしてマスタ管理セクションへ移動する。
3. 業務で使用する選択肢マスタの一覧を確認する。
4. 必要に応じてマスタ値を追加、更新、削除する。
5. 保存後、対象テナントの業務画面に反映する。

### 4-3. SJNETコード設定シナリオ

1. 管理権限を持つ利用者がテナント設定を開く。
2. SJNETコード設定セクションへ移動する。
3. SJ-NETの代理店コードと担当者の対応一覧を確認する。
4. 未登録の代理店コードを新たに追加する、または既存の対応を更新・無効化する。
5. 保存後、次回のSJNET取込から担当者が自動設定されるようになる。

## 6. 通知設定セクション

### 6-1. 目的

満期通知や事故リマインドなど、テナント単位で制御したい通知設定を管理する。

### 6-2. 想定設定項目

| 項目 | 内容 |
| --- | --- |
| 満期通知有効フラグ | 満期通知を有効にするか |
| 満期通知開始日数 | 満期日の何日前から通知するか |
| 満期通知間近日数 | 間近通知を何日前で扱うか |
| 事故リマインド有効フラグ | 事故リマインドを有効にするか |
| 通知先種別 | LINE WORKS などの通知先種別 |
| 通知先情報 | Webhook URL 等の通知先情報 |
| 通知本文設定 | 必要に応じて定型文や件名ルール |

※ 実際の保持項目に応じて調整する。

### 6-3. 主操作

| 操作 | 内容 |
| --- | --- |
| 保存 | 通知設定を保存する |
| 入力値リセット | 変更前の状態へ戻す |

### 6-4. 動作

* 初期表示時に対象テナントの通知設定を表示する。
* 利用者が値を変更して保存すると、対象テナントの設定を更新する。
* 保存成功時はメッセージを表示する。
* 入力不正がある場合は保存しない。

### 6-5. 表示上の考え方

* 満期通知と事故通知をグループ分けして表示する。
* 通知先設定は誤入力しやすいため、ラベルを明確にする。
* 有効フラグがオフの場合、関連項目を非活性にしてよい。

## 7. マスタ管理セクション

### 7-1. 目的

テナントごとに管理したい業務選択肢を、画面から追加・更新・削除できるようにする。

### 7-2. 想定管理対象

| マスタ種別 | 内容 |
| --- | --- |
| 種目 | 保険種目などの選択肢 |
| 営業担当者 | 営業担当の選択肢 |
| 事務担当者 | 事務担当の選択肢 |
| 支店 | 支店の選択肢 |
| ステータス候補 | 一覧や編集で使う状態選択肢 |

※ 実際に管理対象とするマスタは実装仕様に合わせて調整する。

### 7-3. 画面構成

マスタ管理セクション内は、管理対象ごとに一覧と編集領域を持つ構成とする。

| 項目 | 内容 |
| --- | --- |
| マスタ種別選択 | 管理したいマスタ種別を選ぶ |
| 一覧表示 | 登録済みマスタ値を一覧表示する |
| 追加入力欄 | 新規値を追加する |
| 編集導線 | 既存値を編集する |
| 削除導線 | 既存値を削除する |
| 表示順 | 必要に応じて並び順を管理する |
| 有効フラグ | 必要に応じて利用可否を管理する |

### 7-4. 主操作

| 操作 | 内容 |
| --- | --- |
| 追加 | 新しいマスタ値を登録する |
| 更新 | 既存のマスタ値を更新する |
| 削除 | 既存のマスタ値を削除する |
| 並び順変更 | 表示順を変更する |

### 7-5. 動作

* 初期表示時に選択中マスタ種別の一覧を表示する。
* 追加、更新、削除後は一覧を再表示する。
* 他画面で利用中の値の扱いは実装方針に従う。
* 削除できない条件がある場合はメッセージ表示する。

### 7-6. 表示上の考え方

* 管理対象が切り替わっても操作方法が大きく変わらないようにする。
* 業務利用頻度の高いマスタは上位に置く。
* 有効フラグ運用がある場合、物理削除より無効化を優先してよい。

## 8. SJNETコード設定セクション

### 8-1. 目的

SJ-NETからダウンロードする満期一覧データに含まれる「代理店コード」と、本システムのユーザーを紐づける。  
この設定がないと、SJNET取込時に契約・満期案件の担当者が自動設定されない。

**初回SJNET取込前に、全代理店コードの登録を完了させることを強く推奨する。**

### 8-2. 代理店コードについて

SJ-NETの満期一覧データには「代理店コード」（8桁、例: N8559000）が含まれており、担当者1人に1コードが割り当てられている。このコードはSJ-NET上の識別子であり、システムのユーザーIDとは独立している。本セクションでその対応を管理する。

### 8-3. 一覧表示項目

| 項目 | 内容 |
| --- | --- |
| 代理店コード | SJNETの代理店コード（例: N8559000） |
| SJNET担当者名 | SJNETデータに記載の担当者名（参考表示） |
| 対応ユーザー | 本システムのユーザー名 |
| 有効 | 有効フラグ。無効の場合は取込時に担当者未設定となる |
| 備考 | 特殊コードの説明等 |
| 操作 | 編集・無効化 |

### 8-4. 登録・編集フォーム

| 項目 | 入力種別 | 必須 | 備考 |
| --- | --- | --- | --- |
| 代理店コード | テキスト | ○ | 重複不可。SJNETデータの列44の値を入力 |
| SJNET担当者名 | テキスト | - | SJNETデータの列43の値を入力。確認用のみ、マッピングには使用しない |
| 対応ユーザー | セレクト | ○ | テナント所属ユーザーから選択 |
| 有効フラグ | チェックボックス | ○ | デフォルトON |
| 備考 | テキスト | - | 特殊コード等の説明 |

### 8-5. 主操作

| 操作 | 内容 |
| --- | --- |
| 追加 | 新しい代理店コードとユーザーの対応を登録する |
| 編集 | 既存の対応を更新する |
| 無効化 | 有効フラグをOFFにする。コードは保持したまま取込時の自動設定を停止する |
| 削除 | 対応を物理削除する。過去の取込履歴への影響はない |

### 8-6. 動作

* 初期表示時に登録済みの代理店コード一覧を表示する。
* 有効なコードのみを上位に表示し、無効コードは下部に灰色表示する。
* 代理店コードの重複登録はバリデーションで弾く。
* 追加・編集後は一覧を再表示する。
* SJNET取込結果に「マッピング未登録コード」が表示された場合、本セクションから登録する。

### 8-7. SJNET取込との連携

SJNET取込時に本セクションの設定を参照し、以下を自動設定する。

| 設定先 | フィールド | 動作 |
| --- | --- | --- |
| t_contract | sales_user_id | マッピングで解決したuser_id。既に設定済みの場合は上書きしない |
| t_renewal_case | assigned_user_id | マッピングで解決したuser_id。既に設定済みの場合は上書きしない |

代理店コードが未登録または無効化済みの場合、担当者は設定されず取込は続行する（エラー扱いにしない）。  
取込結果に「マッピング未登録: N件」として表示される。

### 8-8. 表示上の考え方

* 有効コードと無効コードを視覚的に区別する（有効: 通常表示、無効: グレーアウト）。
* 「SJNET担当者名」と「対応ユーザー」の両方を並べて表示し、対応関係が直感的に確認できるようにする。
* 代理店コードはそのまま表示し、変換・省略しない。

## 10. 画面動作

### 10-1. 初期表示

* ログイン中ユーザーの所属テナントに対応する設定を表示する。
* 画面上部からテナント概要、通知設定、マスタ管理、SJNETコード設定の順で表示する。
* テナント名を明示し、どのテナント設定を編集しているか分かるようにする。

### 10-2. セクション移動時

* 画面遷移や表示切替は行わず、スクロールで上下のセクションを確認する。
* 未保存変更がある場合の扱いは実装方針に従う。

### 10-3. 通知設定保存時

* 入力チェック後、対象テナントの通知設定を保存する。
* 保存成功時は成功メッセージを表示する。
* 保存失敗時はエラーメッセージを表示する。

### 10-4. マスタ追加・更新・削除時

* 入力チェック後、対象テナントのマスタを更新する。
* 処理後は一覧を再取得して表示する。
* 他データとの整合上、削除できない場合は理由をメッセージ表示する。

### 10-5. SJNETコード追加・編集時

* 代理店コードの重複チェックを行う。
* 対応ユーザーが有効なテナント所属ユーザーであることを確認する。
* 保存成功時は一覧を再表示する。

## 11. 本画面の役割整理

### 11-1. テナント設定の役割

本画面の役割は、テナント単位で業務運用に関わる設定を変更できるようにすることである。

ここで扱うものは以下である。

* 通知の有効・無効
* 通知条件
* 通知先情報
* 業務で使う選択肢マスタ
* SJNETの代理店コードとユーザーの対応

### 11-2. ホームとの違い

ホームは状況確認と導線の画面である。  
本画面は、運用設定そのものを変更する管理画面である。

### 11-3. 分割画面にしない理由

通知設定・マスタ管理・SJNETコード設定はどれもテナント単位の設定であり、利用者視点では同じカテゴリの操作であるため、同一画面内を縦スクロールで連続して扱う。

## 14. 権限・認可

* ログイン済みユーザーのみ表示可能とする。
* 表示・更新対象は所属テナントの設定に限定する。
* 本画面は管理機能として扱い、編集権限は管理権限者に限定することを基本とする。
* 初期実装で参照のみを広く許容する場合でも、更新権限は管理権限者に限定することが望ましい。
* 編集権限がない場合は参照のみ、または画面自体を非表示としてよい。
* 権限差分は通常業務の主導線ではなく、管理機能の公開範囲として扱う。

---

## 15. スマホ表示方針

* セクション見出しは視認しやすいサイズで表示する。
* 通知設定は縦積みフォームを基本とする。
* マスタ管理・SJNETコード設定は一覧を簡略化し、必要に応じて横スクロールを許容する。
* 保存ボタンは画面下部でも操作しやすい位置に置く。

## 16. 例外・考慮事項

### 16-1. 通知先情報が未設定の場合

* 未設定状態を明示する。
* 通知有効化時に必須入力チェックを行ってよい。

### 16-2. マスタ値が他画面で利用中の場合

* 物理削除ではなく無効化を優先してよい。

### 16-3. 複数テナント所属の場合

* 初期仕様では主所属テナントのみ編集対象とする。
* 将来的に切替が必要になった場合は別途検討する。

### 16-4. 未保存状態で画面を離れる場合

* 必要に応じて確認ダイアログを表示してよい。

### 16-5. SJNETコードが未登録のまま取込を実行した場合

* 取込はエラーにならず続行する。
* 取込結果に未解決コードと件数を表示し、本セクションへの誘導リンクを表示する。
* 担当者未設定の契約・満期案件は、満期案件一覧の担当者フィルタで「未設定」として確認できる。

## 18. 非機能要件上の留意点

* テナントをまたぐ更新が起きないこと。
* 通知先情報など誤入力時の影響が大きい項目は、入力補助やバリデーションを設けること。
* マスタ更新後、業務画面への反映タイミングを明確にすること。
* SJNETコード設定は取込処理の前提設定であるため、変更後は即時反映されることを明示すること。
* PC とスマホの両方で操作できること。
