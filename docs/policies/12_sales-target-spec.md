# 目標管理仕様

## 1. 位置づけ

本文書は、`t_sales_target` を用いた目標管理の業務ロジックを定義する。
目標値の登録・更新は `docs/screens/tenant-settings.md` で扱い、
本文書は「目標と成績をどう突合するか」「どの画面にどう表示するか」を扱う。

## 2. 対象テーブル

- `t_sales_target`（目標値）
- `t_sales_performance`（成績値）

## 3. 目標の粒度

`t_sales_target` は以下の3軸で目標を保持する。

| 軸 | 値 |
|---|---|
| 期間軸 | 年度目標（target_month IS NULL）/ 月次目標（target_month 1-12） |
| 担当者軸 | チーム全体目標（staff_user_id IS NULL）/ 担当者別目標（staff_user_id 指定） |
| 種別軸 | premium_non_life / premium_life / premium_total / case_count |

UNIQUE KEY: `(fiscal_year, target_month, staff_user_id, target_type)`

## 4. 目標種別（target_type）の使い分け

| target_type | 用途 | 集計対象 | 運用状況 |
|---|---|---|---|
| `premium_non_life` | 損保保険料目標 | `t_sales_performance.source_type = 'non_life'` の premium_amount 合計 | 運用中 |
| `premium_life` | 生保保険料目標 | `t_sales_performance.source_type = 'life'` の premium_amount 合計 | 運用中 |
| `premium_total` | 保険料合計目標 | 全件の premium_amount 合計（source_type 問わず） | **廃止**（将来拡張用として DDL には残置。新規レコードは作成しない） |
| `case_count` | 件数目標 | 全件の件数（COUNT） | 未使用（将来拡張） |

### 4-1. ホーム画面の年度目標

ホーム画面の「年度目標」は損保・生保の 2 種別を別タイルで表示し、全体タイルでは `premium_non_life + premium_life` の合計を使用する。  
`premium_total` は参照しない。

## 5. 目標と成績の突合ロジック

### 5-1. 基本原則

- 目標と成績は `fiscal_year` と `staff_user_id` の組み合わせで突合する
- 成績の期間判定は `settlement_month` を基準にする
  - 理由: 成績管理の業務慣習では、成績は「精算月」で集計される
  - `performance_date`（計上日）は参考値であり、集計の主軸にはしない
- 成績の集計は `is_deleted = 0` のレコードのみを対象とする
- `settlement_month IS NULL` のレコードは集計対象から除外する

### 5-2. 年度の定義

- 年度 = 4月1日〜翌年3月31日
- `fiscal_year` の値は年度開始年の西暦を使う
  - 例: `fiscal_year = 2025` は 2025年4月〜2026年3月

### 5-3. `settlement_month` の年度判定

`t_sales_performance.settlement_month` は `YYYY-MM` 形式の文字列。
以下のルールで年度を判定する:

- `settlement_month` の月が 4月〜12月 → その年の年度
- `settlement_month` の月が 1月〜3月 → 前年の年度
- `settlement_month IS NULL` のレコードは集計対象から除外する

例:
- `settlement_month = '2025-05'` → 2025年度
- `settlement_month = '2026-02'` → 2025年度
- `settlement_month = '2026-04'` → 2026年度

### 5-5. 年度累計と YTD の集計範囲

年度累計および月次推移テーブルの「年間」列は、以下のルールで計算する。

#### 現年度選択時

- 集計範囲: 選択年度の4月〜現在月まで
- 未来月は集計対象外
- 前年同期も前年度の4月〜同じ月までの範囲で集計（YTD 同士の比較）
- ラベル表示例: 「2026年度 4月〜4月」

#### 過去年度選択時

- 集計範囲: 選択年度の全期間（4月〜翌年3月）
- 前年同期も前年度の全期間で集計
- 過去年度の全月は「確定済み」として扱う
- ラベル表示例: 「2025年度（4月〜3月）」

#### 月次推移テーブルの「年間」列

| 行 | 現年度 | 過去年度 |
|---|---|---|
| 成績 | 4月〜当月の合計（YTD） | 4月〜3月の合計 |
| 前年 | 前年度の4月〜当月の合計（YTD） | 前年度の4月〜3月の合計 |
| 前年差 | 成績YTD - 前年YTD | 全期間差 |
| 目標 | 4月〜当月の月次目標合計 | 4月〜3月の月次目標合計 |

重要: 「年間」列の前年は前年度全期間ではなく、同じ YTD 範囲で比較する。

### 5-6. 未来月の表示ルール

月次推移テーブル・グラフにおいて、未来月は全ての行で「—」とする。

| 項目 | 未来月の表示 |
|---|---|
| 当年度成績 | — |
| 前年 | — |
| 前年差 | — |
| 月次目標 | — |
| グラフ棒 | 描画しない（当年度・前年ともに） |

「未来月」の定義:
- 現在年度の選択時: 現在月より後の月（年度内の月順 4〜3 で判定）
- 過去年度の選択時: 未来月は存在しない（全月が確定済み）

理由: 「未到来の月」と「ゼロ成績」を明確に区別するため。
前年データがあっても未来月は比較対象としない。
前年の数値を見せると「比較対象があるのに成績がない」印象を与えるため、
ユーザーが誤解しないよう未来月は全行「—」とする。

## 6. 担当者軸の突合ルール

### 6-1. 担当者の特定

成績の担当者は `t_sales_performance.staff_id` で判定する。

### 6-2. ホーム画面の担当者ドロップダウンと目標の対応

| ドロップダウン選択 | 集計対象の成績 | 参照する目標 |
|---|---|---|
| 自分 | staff_id = ログインユーザー | staff_user_id = ログインユーザー |
| 全体 | 全件 | staff_user_id IS NULL（チーム全体目標） |
| 特定ユーザー | staff_id = 選択ユーザー | staff_user_id = 選択ユーザー |

### 6-3. 目標未登録時の表示

該当する目標レコードが存在しない場合、ホーム画面では「目標未設定」と表示する。
エラー扱いにはしない。

- 例1: 「自分」を選択したが自分の担当者別目標が未登録 → 「目標未設定」
- 例2: 「全体」を選択したがチーム全体目標が未登録 → 「目標未設定」

チーム全体目標と担当者別目標はそれぞれ独立して管理される。
担当者別目標の合計がチーム全体目標と一致する保証はない。

## 7. 月次目標と年度目標の整合性

### 7-1. 基本方針

月次目標（`target_month` 1-12）と年度目標（`target_month IS NULL`）は
**それぞれ独立に登録する**。システムは両者の整合性を強制しない。

理由:
- 実務的に月次の波動を許容したい（繁忙期・閑散期）
- 月次目標の合計 ≠ 年度目標 になるケースを許容する

### 7-2. 表示時の優先順位

ホーム画面の月次推移テーブルで「目標」行を表示する場合:

- 該当月の月次目標が登録されていればそれを表示
- 月次目標が未登録の場合は「—」を表示
- 年度目標から自動按分はしない

年度目標は月次推移テーブルには表示せず、年度累計カードに表示する。

## 8. 目標進捗の計算式

### 8-1. 年度進捗
年度進捗率 = (年度累計成績 / 年度目標) × 100

- 小数点以下は切り捨てまたは四捨五入（実装統一）
- 目標が 0 または未設定の場合は「目標未設定」と表示
- 100% 超過時もそのまま表示（120% など）

### 8-2. 月次進捗
月次進捗率 = (当月成績 / 当月目標) × 100

- 当月の定義: `settlement_month = YYYY-MM` （当月）
- 目標が 0 または未設定の場合は「目標未設定」と表示

### 8-3. 期間按分はしない

例えば4月15日時点での進捗計算で「目標 × 15/30」のような期間按分はしない。
月末時点で初めて100%に到達するイメージではなく、月内のどの時点でも
同じ目標値に対する進捗率を表示する。

## 9. 前年比の計算式

### 9-1. 前年同期累計
前年同期累計 = 前年度の4月〜当月（月単位）の累計成績

- 月単位の粒度で集計する（日単位の按分はしない）
- 例: 今年度 2026年4月7日 時点 → 前年同期は 2025年4月〜2025年4月 の前年度月次累計

### 9-2. 前年比
前年比(%) = (今年度累計 / 前年同期累計) × 100

- 前年同期が 0 の場合は「—」を表示（0 除算回避）
- 前年同期が未登録（レコード無し）の場合も「—」を表示

### 9-3. 前年差
前年差 = 今年度累計 - 前年同期累計

月次推移テーブルの「前年差」列に表示する。

## 10. 表示単位

### 10-1. 金額表示

- DB値: 円単位（整数）
- 画面表示: 千円単位に変換して表示
- 変換式: `円 ÷ 1000`（小数点切り捨て）
- 単位表記: 「千円」

### 10-2. 件数表示

- DB値: 整数
- 画面表示: そのまま
- 単位表記: 「件」

## 11. 画面別の表示仕様

### 11-1. ホーム画面 成績サマリ

年度累計カードには「全体 / 損保 / 生保」の 3 タイルを横並びで表示する。各タイルに以下を表示する:

| 表示項目 | 全体タイル | 損保タイル | 生保タイル |
|---|---|---|---|
| 金額 | `source_type IN ('non_life','life')` の premium_amount 合計 | `source_type = 'non_life'` の合計 | `source_type = 'life'` の合計 |
| 件数 | 同上の件数 | 同上の件数 | 同上の件数 |
| 前年比 | 9-2 のルール。分母 0 は `—` | 同左 | 同左 |
| 年度目標 | `premium_non_life + premium_life` の合計 | `premium_non_life` | `premium_life` |
| 達成率 | （金額 / 目標）× 100。目標未設定時は「目標未設定」 | 同左 | 同左 |

月次推移テーブルには以下を表示する（業務区分別の内訳は出さず、全体のみを従来通り表示）:

| 行 | 計算方法 |
|---|---|
| 成績（今年度） | 各月の settlement_month に該当する成績合計 |
| 前年 | 前年度の同月の成績合計 |
| 前年差 | 成績 - 前年 |
| 累積達成率 | 4月〜当該月までの累積成績 / 年度目標 × 100。目標未設定または未来月は「—」 |

### 11-2. 成績管理一覧・詳細

目標関連の表示はしない。成績一覧・詳細は「成績そのもの」を扱う画面であり、
目標との比較はホーム画面の責務とする。

## 12. 権限・認可

- 目標の登録・更新はテナント管理者のみ（既存仕様）
- 目標の閲覧（ホーム画面での表示）は全ログインユーザー
- 他ユーザーの目標値もホーム画面から参照できる（担当者ドロップダウンで切替）
