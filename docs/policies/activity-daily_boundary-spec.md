# 活動一覧 / 日報ビュー 導線境界仕様

## 1. 問題

活動一覧（`activity/list`）と日報ビュー（`activity/daily`）の導線は
「一覧の日付リンクから日報ビューへ」という一方向の記述のみであり、
以下が未定義であった。

- 今日以外の日付でも日報ビューを開けるか
- 日報ビューから活動一覧への戻り方
- 日報ビューを直接 URL 指定で開けるか

## 2. 仕様の確定

### 2-1. 日報ビューは「任意の日付」で開ける

日報ビューは `activity/daily?date=YYYY-MM-DD&staff_user_id=N` で任意の日付を指定できる。

`date` パラメータが省略された場合はサーバー日付（今日）をデフォルトとする。
`staff_user_id` が省略された場合はログインユーザー自身をデフォルトとする。

過去日・未来日のいずれも指定可能とする。
未来日は活動0件として表示し、エラーとしない。

### 2-2. 活動一覧から日報ビューへの遷移

活動一覧の `activity_date` 列に日付リンクを設ける。
リンク URL は `activity/daily?date={activity_date}&staff_user_id={staff_user_id}` とする。

`staff_user_id` は一覧で現在フィルタ中の担当者を引き継ぐ。
「全員」でフィルタ中の場合は `staff_user_id` を省略し、日報ビュー側でデフォルト（ログインユーザー）を使う。

### 2-3. 日報ビューから活動一覧への戻り

日報ビューのヘッダに「活動一覧へ戻る」リンクを設ける。
遷移元の検索条件・ページ・表示件数を保持する URL パラメータを維持する（`Referer` ではなく URL パラメータで保持）。

### 2-4. 日報ビューの直接 URL アクセス

`activity/daily` は直接 URL 指定でアクセス可能とする。
ログイン必須。未認証の場合はログイン画面へリダイレクトする。

### 2-5. 日報ビューでの日付切替

日報ビュー上に「前日」「翌日」のナビゲーションリンクを設ける。
当日分は「今日」ラベルで今日の日報ビューに戻れるようにする。

### 2-6. 管理者の担当者切替

管理者ロール（admin）は日報ビュー上で担当者を切り替えられる。
切替後の URL は `activity/daily?date=...&staff_user_id=N` となる。

一般ロール（member）は自分の日報のみ参照できる。
`staff_user_id` に他ユーザーのIDを指定してもアクセスを拒否し、
ログインユーザー自身の日報にリダイレクトする。

## 3. 画面遷移まとめ

| 操作 | 遷移先 |
|---|---|
| 活動一覧 → 日付リンク | `activity/daily?date={date}&staff_user_id={id}` |
| 日報ビュー → 活動一覧へ戻る | `activity/list`（検索状態を保持） |
| 日報ビュー → 前日 | `activity/daily?date={前日}&staff_user_id={id}` |
| 日報ビュー → 翌日 | `activity/daily?date={翌日}&staff_user_id={id}` |
| 日報ビュー → 今日 | `activity/daily`（パラメータなし） |
| 日報ビュー → 活動詳細 | `activity/detail?id={id}` |
