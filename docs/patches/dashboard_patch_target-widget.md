# SCR-DASHBOARD 目標対比ウィジェット 未設定時表示仕様

## 1. 問題

`t_sales_target` を参照する目標対比ウィジェットは、
テナント設定の目標管理セクション（Phase 設定A）が未完了の状態では
目標値が1件も登録されていない。

この状態でダッシュボードにアクセスした場合の表示仕様が未定義であった。

## 2. 表示仕様の確定

### 2-1. 目標値が1件も登録されていない場合

目標対比ウィジェットを非表示にする。
エラーメッセージは出さない。

目標が未設定であることをダッシュボード上でユーザーに伝える必要はない。
目標管理はテナント設定から行うものであり、ダッシュボードで案内する責務を持たせない。

### 2-2. 年度目標のみ登録されていて月次目標がない場合

月次の対比表示は行わず、年度集計と年度目標の対比のみを表示する。

### 2-3. チーム全体目標のみ登録されていて担当者別目標がない場合

チーム全体の実績と全体目標の対比のみを表示する。
担当者別の目標対比行は表示しない。

### 2-4. 目標はあるが実績が0件の場合

実績 0 / 目標 X の形式で表示する。
0件であることをエラーとして扱わない。

## 3. ウィジェット表示条件のまとめ

| 状態 | 表示 |
|---|---|
| 目標0件 | ウィジェット非表示 |
| 年度目標のみ | 年度対比のみ表示 |
| 月次目標あり | 当月対比を表示 |
| 担当者別目標なし | チーム全体対比のみ表示 |
| 担当者別目標あり | 担当者行を追加表示 |

## 4. 実装上の注意

ウィジェット表示の有無は `t_sales_target` の件数チェックで制御する。

```sql
SELECT COUNT(*) FROM t_sales_target
WHERE fiscal_year = :fiscal_year
  AND is_deleted = 0
```

0件の場合はウィジェットのHTMLブロックごと出力しない。
クエリの失敗はエラーログに記録し、ウィジェット非表示として扱う（ダッシュボード全体はエラーにしない）。

## 5. 当月の計算基準

`target_month` は 1〜12 の月を表す（会計年度ではなく暦月）。
当月の判定はサーバーの日付（`DATE('now')`）から取得する。

年度（`fiscal_year`）は4月始まりの会計年度とし、
現在が2026年1〜3月であれば `fiscal_year = 2025` を参照する。

```
fiscal_year の算出: 現在月 >= 4 ? 現在年 : 現在年 - 1
```
