1. 初回ログイン

本システムは Google アカウント による認証と、スマートフォンの認証アプリを使った 二段階認証（2FA） でログインします。初回のみ以下の手順でセットアップしてください。

1-1. Google Authenticator をインストールする

二段階認証用のアプリとして Google Authenticator を使用します。お持ちのスマートフォンにインストールしてください。

iPhone（iOS）App Store で「Google Authenticator」を検索してインストール
AndroidGoogle Play で「Google Authenticator」を検索してインストール

補足 Microsoft Authenticator や 1Password など、他の TOTP 対応アプリでも利用できます。

1-2. Google でログインする

  1. ブラウザで本システムの URL を開きます。
  2. ログイン画面の「Google でログイン」ボタンをクリックします。
  3. Google のアカウント選択画面で、管理者から案内されたアカウントを選択します。
  4. 初回はアプリへのアクセス許可の同意画面が表示されるので「許可」をクリックします。

注意 ログインできない場合は、管理者が本システムにあなたのアカウントを登録しているか確認してください。登録されていないアカウントではログインできません。

1-3. 二段階認証の初回設定

  1. Google ログイン成功後、画面に QR コード と 16 桁程度のキーが表示されます。
  2. スマートフォンで Google Authenticator を開き、「+（追加）」→「QR コードをスキャン」を選びます。
  3. 画面の QR コードを読み取ります。アプリにアカウントが追加され、6 桁の数字が 30 秒ごとに更新表示されます。
  4. ブラウザの入力欄に、今表示されている 6 桁のコード を入力して「確認」をクリックします。
  5. 設定完了メッセージが表示されれば完了です。

重要 Google Authenticator の登録情報は、スマホを紛失・機種変更すると再取得できません。管理者にリセットしてもらう必要があります。機種変更時は事前にバックアップ機能を使うか、管理者に相談してください。

1-4. 2 回目以降のログイン

  1. 「Google でログイン」で Google 認証を通します（多くの場合ワンクリック）。
  2. 6 桁のコード入力画面が表示されたら、Google Authenticator の最新コードを入力します。
  3. ホーム画面が表示されればログイン完了です。

2. ホーム画面

ログイン後に最初に表示される画面です。画面は大きく 3 つのエリアで構成されています。

2-1. 要確認エリア（上段）

今日すぐに対応すべき件数を赤色で目立たせて表示します。

  • 満期カード: 「対応遅れ」件数と「7 日以内」の満期件数。カードをクリックすると、満期一覧が「7 日以内」で絞り込まれた状態で開きます。
  • 事故カード: 「高優先度・未完了」の事故件数。カードをクリックすると、事故案件一覧が「高優先度・未完了」で絞り込まれた状態で開きます。

2-2. 業務入口エリア（中段）

日常業務の 3 つの入口カードが並びます。

  • 満期業務: 14 日以内 / 28 日以内 / 60 日以内の件数を表示
  • 事故案件: 優先度別（高 / 中 / 低）の件数を表示
  • 見込管理: 見込ランク別 / 今月成約見込の件数を表示

各カード右上の担当者プルダウンで表示対象を切り替えできます（全体 / 自分 / 特定の担当者）。カード本体をタップすると、選択中の担当者で絞り込まれた一覧が開きます。

2-3. 成績サマリ（下段）

  • 年度タブ: 今年度・前年度・前々年度を切り替え
  • 担当者プルダウン: 全体 / 個別担当者
  • 3 タイル: 全体 / 損保 / 生保の「金額」「件数」「達成率」「前年比」「年度目標」
  • 月次推移表: 4 月〜翌 3 月の月別実績と前年・目標の比較

画面上部のクイックフィルタチップで、よく使う条件をワンタップで切り替えできます。

さらに詳細な条件を指定したい場合は「絞り込み」ボタンから、担当者・顧客名・対応状況などを組み合わせて検索できます。

3-2. 対応状況の更新

  1. 一覧から対象の行をタップして詳細画面を開きます。
  2. 「対応状況の更新」フォームで、事務担当・対応状況・更改方法・手続き方法・完了日・次回対応予定日を入力します。
  3. 「保存」ボタンをクリックして確定します。

3-3. 顧客紐づけ／営業担当の変更

詳細画面のヘッダ部分に「顧客を紐づける」「営業担当を変更」ボタンがあります。未紐づけの満期に顧客を割り当てたり、営業担当者を別のスタッフに変更したりできます。

3-4. SJ-NET CSV 取込

損保ジャパンの SJ-NET からダウンロードした満期データ（CSV）を取り込み、契約と満期案件を一括で登録・更新します。

取込の入口

  1. 満期一覧画面の上部ツールバーにある「CSV 取込」ボタンをクリック。
  2. 「SJNET 満期データ取込」ダイアログが開きます。
  3. CSV ファイルをドラッグ&ドロップするか、クリックしてファイルを選択。
  4. ファイル選択後、「取込を開始」ボタンが有効化されるのでクリック。
  5. 処理が完了すると、同じダイアログ内に結果サマリが表示されます。

対象ファイル

  • SJ-NET から出力された CSV ファイル（文字コードは Shift-JIS / UTF-8 のどちらでも OK）
  • 1 行目がヘッダ行（カラム名）であること
  • カラム名で値を特定するため、列の並び順は任意。不要な列が含まれていても問題ありません

必須カラム

ヘッダ行に以下の 7 つのカラム名が含まれている必要があります。1 つでも欠けていると取込できません。

カラム名用途
顧客名顧客の照合 / 新規登録
保険始期契約情報
保険終期満期日として使用
種目種類契約情報
証券番号契約の重複判定キー
合計保険料契約情報
代理店ｺｰﾄﾞ担当者の自動割当

注意「代理店ｺｰﾄﾞ」の「ｺｰﾄﾞ」は半角カタカナです。SJ-NET からの出力をそのまま使えば問題ありません。

任意カラム

含まれていれば取り込まれ、無くてもエラーにはなりません。

カラム名用途
生年月日顧客照合の補助 / 新規顧客として登録する際に保存
郵便番号新規顧客登録時のみ反映（既存顧客には上書きしない）
住所同上
ＴＥＬ同上
払込方法契約の補助情報
担当者参考として保持（担当者の自動割当には使用しない）

担当者の自動割当

CSV の「代理店ｺｰﾄﾞ」と、設定画面のユーザー管理で登録した代理店コードを照合し、一致したスタッフを担当者として自動設定します。

  • 一致するコードが見つからない場合は、担当者が未設定のまま取り込まれます
  • 取込は中止されず、最後まで処理されます

事前に確認 CSV 取込の前に、CSV に登場する代理店コードがすべて「設定 → ユーザー管理」で登録されているか確認してください。

既存データとの統合

同じ証券番号・満期日の契約がすでにある場合は更新、無い場合は新規登録されます。

  • 既存契約の営業担当が設定済みの場合、CSV で上書きされません（画面で設定した内容が優先されます）
  • 顧客名で既存顧客が見つからない / 複数ヒットする場合は「未紐づけ」として登録され、満期詳細画面から後で手動で紐づけられます
  • 既存顧客の住所・電話・郵便番号は CSV で上書きされません

取込結果の確認

取込完了後、ダイアログ内に以下が表示されます。

  • 処理件数サマリ（新規登録 / 更新 / スキップ / エラー 等）
  • 担当者マッピング結果（解決済み / 未登録コード）
  • 未紐づけやエラー行の一覧（行番号・証券番号・顧客名とあわせて表示）

よくあるエラー

症状対処
「CSV ヘッダが不足しています」必須カラムが揃っているか、カラム名が正確か確認
行がスキップされるその行の証券番号・顧客名・保険終期が空になっていないか確認
顧客が未紐づけになる満期詳細画面から手動で顧客を紐づける
担当者が空のまま取り込まれる設定 → ユーザー管理で代理店コードを登録する
文字化けするSJ-NET から再度出力する

運用のコツ

  • 初回取込前に、全スタッフの代理店コードをユーザー管理に登録しておく
  • 少量のデータで試験取込してから本番の取込を行う
  • 取込後に未紐づけの件数を確認し、必要に応じて手動で顧客を紐づける
  • 同じ CSV を繰り返し取り込んでも重複登録されません（証券番号・満期日で既存判定するため）

4. 事故案件

4-1. 一覧の見方・絞り込み

クイックフィルタ:

  • 高優先度・未完了: 優先度「高」でかつ完了扱いになっていない案件
  • 未完了: 完了扱い以外すべて
  • 自分担当: 自分が担当の案件
  • 完了: 完了扱いの案件

担当者プルダウン・絞り込みダイアログで更に条件を追加できます。

4-2. 案件の登録

  1. 一覧画面右上の「新規登録」ボタンをクリックします。
  2. ダイアログで顧客・事故概要・発生日・優先度（高 / 中 / 低）・担当者を入力します。
  3. 「保存」で登録完了。一覧に追加されます。

4-3. 更新

一覧から対象行をタップして詳細画面を開き、対応状況・担当者・コメントを更新して保存します。コメントは履歴として残ります。

5. 見込案件

5-1. 一覧の見方・絞り込み

事故案件と同様、クイックフィルタ（高優先度・未完了 / 未完了 / 自分担当 / 完了）と担当者プルダウンで絞り込めます。

5-2. 案件の登録・更新

  1. 「新規登録」ボタン → ダイアログで顧客・案件名・案件区分・見込ランク（A / B / C）・想定成約日・担当者を入力。
  2. 詳細画面で対応状況やコメントを更新できます。
  3. 成約／失注の切り替えは「対応状況」で行います。

6. 営業日報

6-1. 活動の記録方法

訪問・電話・メールなど、日々の営業活動を記録します。

  1. メニューから「営業日報」または「活動一覧」を開きます。
  2. 「新規登録」または当日の行の「＋」ボタンで入力フォームを開きます。
  3. 活動日・活動種別（訪問 / 電話 / メール / オンライン / 会議 / 研修 / その他）・用件区分（満期対応 / 新規開拓 / クロスセル提案 / 事故対応 / 見積対応 / 保全対応 / 苦情対応 / その他）・顧客・担当者・備考を入力します。
  4. 「保存」で登録完了。

補足 活動種別・用件区分の選択肢は管理者が設定画面で追加・変更できます。

7. 成績管理

7-1. 成績の確認方法

  • ホーム画面の成績サマリ: 年度・担当者ごとの金額と件数、達成率、前年比、月次推移
  • 成績一覧画面: 明細レベルでの確認。年度・月・担当者・業務区分（損保 / 生保）で絞り込み可能

7-2. 成績の登録・編集

  1. 成績一覧画面で「新規登録」から 1 件ずつ、または「一括登録」から複数件をまとめて登録します。
  2. 顧客・契約・成績区分（新規 / 継続 / 追加 / 変更 / 解約控除）・保険料・精算月・担当者・販売チャネルなどを入力します。
  3. 既存明細は行をタップして詳細から編集・削除できます。

8-1. 顧客情報の確認

一覧の列構成:

  1. 「新規登録」ボタンをクリック。
  2. 顧客区分（個人 / 法人）を選択。

9. 設定

権限 設定画面の操作は管理者のみ可能です。管理者権限のないユーザーがアクセスするとホーム画面に戻されます。

左ナビから以下の項目を選択できます。

組織

ユーザー管理

代理店に所属するスタッフを管理します。編集ボタンからダイアログを開き、以下を設定します。

  • 業務表示名: 画面上の表示名
  • システムロール: 管理者設定画面も操作可能 / メンバー通常ユーザー
  • 代理店コード: CSV 取込時に担当者を自動割当するためのキー
  • 業務ロール: 営業 / 事務（複数選択可）

満期管理

満期対応で使用する各種マスタを管理します。いずれも追加・編集・削除・有効 / 無効切替が可能で、「無効」にした項目は新規登録時の選択肢から外れますが既存データには影響しません。

  • 種目: 保険種目（自動車、火災 など）
  • 手続き方法: 対面・郵送・対面ナビ など
  • 更改方法: 対面 / 郵送 / 電話募集 など
  • 対応状況: 未対応・SJ 依頼中・書類作成済 など。「完了扱い」にチェックを入れた状況は、ダッシュボード集計・通知バッチ・リマインダーから除外されます

事故管理

対応状況

事故案件の対応状況マスタ。追加・編集・削除・有効 / 無効切替・完了扱いチェックが可能です。

見込案件

対応状況

見込案件のステータスマスタ。追加・編集・削除・有効 / 無効切替が可能です。

営業活動

  • 活動種別: 訪問 / 電話 / メール など
  • 用件区分: 満期対応 / 新規開拓 / 事故対応 など

いずれも追加・編集・削除・有効 / 無効切替ができます。

運用

通知設定

LINE WORKS への通知送信を設定します。

  • 満期通知 有効 / 無効: チェックで切替
  • Webhook URL: LINE WORKS トークルームの Webhook URL を入力
  • 早期通知: 満期日の何日前から通知するかを設定（デフォルト 28 日前）
  • 直前通知: 同上（デフォルト 15 日前）
  • 日数の選択肢は 7 / 14 / 21 / 28 / 30 / 45 / 60 日
  • 事故通知: 満期通知とは別に設定可能。Webhook URL を空欄にすると満期通知と同じ URL を使用

目標管理

  • 対象年度をドロップダウンで選択
  • チーム全体目標: 損保目標・生保目標を円単位で入力（合計は自動計算）
  • 担当者別目標: 各ユーザーごとに損保・生保目標を個別設定
  • 空欄で保存するとその目標は未設定になります

10. よくあるトラブルと対処法

症状対処
ログインできない 管理者にあなたの Google アカウントが登録されているか確認。会社アカウントと個人アカウントを取り違えていないかも確認してください。
二段階認証の 6 桁コードが合わない スマートフォンの時刻が自動設定になっているか確認してください。時刻が数秒ずれていると認証コードが一致しません（iOS: 設定 → 一般 → 日付と時刻 → 自動設定 ON / Android: 設定 → システム → 日付と時刻 → 自動）。
スマホを機種変更 / 紛失して Authenticator が開けない 管理者に連絡して二段階認証をリセットしてもらってください。リセット後、新しいスマホで再度 QR コード登録が必要です。
LINE WORKS に通知が飛ばない 設定 → 通知設定で「有効」になっているか、Webhook URL が正しいかを確認。対応状況が「完了扱い」になっている案件には通知されません。早期通知 / 直前通知の日数が保存されているかも確認してください。
画面が古い・表示がおかしい ブラウザを強制リロード（Windows: Ctrl+Shift+R / Mac: Cmd+Shift+R）で最新版を取得してください。
CSV 取込エラー（満期一覧） 設定 → ユーザー管理で、CSV 内の代理店コードと一致するユーザーが登録されているか確認してください。
設定画面が開けない あなたのアカウントが「管理者」ロールでない可能性があります。管理者に権限付与を依頼してください。
通知設定画面で「早期通知・直前通知」が表示されない システムの初期設定が完了していない可能性があります。管理者にお問い合わせください。
